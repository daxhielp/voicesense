<objective>
## Goal
Replace `librosa.load()` with `soundfile.read()` + `scipy.signal.resample_poly()` in `preprocess_audio()`, dropping step 2 from ~28,000 ms to ~50 ms.

## Purpose
02-01 confirmed `librosa.load()` with resampling accounts for 94.5% of total inference time (27,999 ms of 29,639 ms). The resampy fallback for the 48,000→22,050 Hz ratio (320:147) is the cause. soundfile decodes WAV in milliseconds; scipy.signal.resample_poly handles rational ratios efficiently via polyphase filtering. Both packages are already installed (soundfile is a librosa dependency; scipy is in requirements.txt).

## Output
- `backend/inference.py` — `preprocess_audio()` rewritten; no new dependencies
- Profiler re-run confirming step 2 drops to <200 ms
</objective>

    ### 2. Replace librosa.load() in preprocess_audio()

    Find this block in preprocess_audio():
    ```python
    y, _ = librosa.load(io.BytesIO(wav_bytes), sr=SAMPLE_RATE, mono=True)
    ```

    Replace it with:
    ```python
    # Decode WAV with soundfile (fast — no resampling here)
    y_raw, native_sr = sf.read(io.BytesIO(wav_bytes), dtype="float32", always_2d=False)

    # Convert stereo → mono if needed
    if y_raw.ndim > 1:
        y_raw = y_raw.mean(axis=1)

    # Resample to SAMPLE_RATE (22050 Hz) if needed using polyphase filter
    if native_sr != SAMPLE_RATE:
        g = gcd(native_sr, SAMPLE_RATE)
        y = scipy.signal.resample_poly(y_raw, SAMPLE_RATE // g, native_sr // g)
    else:
        y = y_raw
    ```
