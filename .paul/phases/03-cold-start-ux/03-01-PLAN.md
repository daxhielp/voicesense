<objective>
## Goal
Make Render free-tier cold starts invisible (or at least graceful) through three coordinated UX changes: keep-alive pings to prevent sleep, a dedicated error state with retry for timeouts, and time-aware processing messages that set honest expectations.

## Purpose
After Phase 2, inference takes ~1.2s locally but ~5–10s on Render shared CPU. Cold starts add another 30–60s. Without UX improvements, users hit a blank spinner for up to a minute then see a generic error — with no explanation and no path forward. These three changes make that experience dramatically better with minimal code.

## Output
- `frontend/src/api/predict.ts` — exports `API_BASE` for reuse
- `frontend/src/App.tsx` — keep-alive hook, dedicated error state, processing timer
- `frontend/src/components/ErrorView.tsx` — new component: error message + retry button
</objective>

## AC-1: Keep-alive prevents Render sleep between requests
```gherkin
Given the app is open in the browser
When the page loads
Then a GET /health ping is sent immediately
  and repeated every 4 minutes while the page is open
  and ping failures are silently swallowed (no UI impact)
```

## AC-2: Timeout produces a dedicated error view with retry button
```gherkin
Given the user has recorded audio and the server times out (or returns 5xx)
When the axios request rejects
Then the app transitions to an 'error' AppState (not back to 'idle')
  and ErrorView shows a human-readable message explaining the cold-start situation
  and a "Try Again" button returns the user to 'idle' to re-record
```

## AC-3: Processing messages evolve during long requests
```gherkin
Given the app is in 'processing' state
When fewer than 5 seconds have elapsed
Then the message reads "Analyzing your voice..."
When 5–12 seconds have elapsed
Then the message reads "Processing audio..."
When 12–20 seconds have elapsed
Then the message reads "Server warming up, please wait..."
When more than 20 seconds have elapsed
Then the message reads "Almost there..." with elapsed seconds shown
```
