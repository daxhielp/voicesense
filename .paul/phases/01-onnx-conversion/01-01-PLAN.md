<objective>
## Goal
Replace PyTorch with ONNX Runtime in the backend inference pipeline, shrinking the Docker image ~150 MB and eliminating the torch cold-start overhead.

## Purpose
The Render free-tier Docker image currently installs a CPU-only PyTorch wheel (~150 MB) solely to run one model forward pass. Exporting EmotionCNN to ONNX and switching to `onnxruntime` removes that wheel entirely while keeping all other preprocessing unchanged (librosa mel spec pipeline stays identical to training).

## Output
- `backend/export_onnx.py` — one-time local script that produces `model/emotion_model.onnx`
- `model/emotion_model.onnx` — committed ONNX model (~5–8 MB)
- `backend/inference.py` — rewritten: no torch, uses `onnxruntime.InferenceSession`
- `backend/requirements.txt` — torch removed, onnxruntime added
- `backend/Dockerfile` — torch install layer removed
</objective>

    import torch, torch.nn as nn
    class EmotionCNN(nn.Module):
        def __init__(self, num_classes=8):
            super().__init__()
            self.block1 = nn.Sequential(nn.Conv2d(1,32,3,padding=1), nn.BatchNorm2d(32), nn.ReLU(True), nn.MaxPool2d(2))
            self.block2 = nn.Sequential(nn.Conv2d(32,64,3,padding=1), nn.BatchNorm2d(64), nn.ReLU(True), nn.MaxPool2d(2))
            self.block3 = nn.Sequential(nn.Conv2d(64,128,3,padding=1), nn.BatchNorm2d(128), nn.ReLU(True), nn.MaxPool2d(2))
            self.classifier = nn.Sequential(nn.Dropout(0.5), nn.Linear(128*16*16, 256), nn.ReLU(True), nn.Linear(256, num_classes))
        def forward(self, x): x = self.block1(x); x = self.block2(x); x = self.block3(x); return self.classifier(x.view(x.size(0),-1))
    ```

    2. Load the model from MODEL_PATH (`Path(__file__).parent.parent / "model" / "emotion_model.pkl"`)

       - Call torch.load(MODEL_PATH, map_location="cpu", weights_only=False)
       - Restore __main__ in finally block
       - If result is dict, instantiate EmotionCNN and call load_state_dict(obj)

        input_names=["mel_spectrogram"],
        output_names=["logits"],
        dynamic_axes={"mel_spectrogram": {0: "batch_size"}, "logits": {0: "batch_size"}},
    )
    print(f"Exported → {out_path}  ({out_path.stat().st_size / 1e6:.1f} MB)")
    ```

    New singleton loader:
    ```python
    _session: ort.InferenceSession | None = None

    def load_model() -> ort.InferenceSession:
        global _session
        if _session is not None:
            return _session
        logger.info("Loading ONNX model from %s", MODEL_PATH)
        _session = ort.InferenceSession(
            str(MODEL_PATH), providers=["CPUExecutionProvider"]
        )
        logger.info("ONNX session ready, inputs=%s", [i.name for i in _session.get_inputs()])
        return _session
    ```

    New preprocess_audio() — returns np.ndarray (1,1,128,128) float32 instead of torch.Tensor:
    - Keep all existing logic through `mel_db = librosa.power_to_db(mel, ref=np.max)`
    - Replace F.interpolate with scipy.ndimage.zoom:
      ```python
      h, w = mel_db.shape
      zoom_h = IMG_SIZE[0] / h
      zoom_w = IMG_SIZE[1] / w
      mel_resized = scipy.ndimage.zoom(mel_db, (zoom_h, zoom_w), order=1)
      t = mel_resized[np.newaxis, np.newaxis, :, :].astype(np.float32) / NORM_FACTOR
      return t  # (1, 1, 128, 128)
      ```
    - Note: scipy order=1 bilinear is equivalent to PyTorch bilinear for continuous data;
      minor sub-pixel differences are within model noise tolerance.

    New predict() — uses session.run() instead of model():
    ```python
    def predict(audio_bytes: bytes, content_type: str) -> dict:
        session = load_model()
        tensor = preprocess_audio(audio_bytes, content_type)  # numpy (1,1,128,128)

        input_name = session.get_inputs()[0].name
        logits = session.run(None, {input_name: tensor})[0]  # numpy (1,8)

        # Numerically stable softmax
        logits = logits.squeeze()  # (8,)
        exp_l = np.exp(logits - logits.max())
        probs = exp_l / exp_l.sum()

        pred_idx = int(probs.argmax())
        pred_name = EMOTION_LABELS[pred_idx]

        probabilities = {
            EMOTION_LABELS[i]: round(float(probs[i]) * 100, 2)
            for i in range(8)
        }
        probabilities_sorted = dict(
            sorted(probabilities.items(), key=lambda x: x[1], reverse=True)
        )
        return {
            "emotion":       pred_name,
            "confidence":    round(float(probs[pred_idx]) * 100, 2),
            "probabilities": probabilities_sorted,
            "color":         EMOTION_META[pred_name]["color"],
            "emoji":         EMOTION_META[pred_name]["emoji"],
        }
    ```
